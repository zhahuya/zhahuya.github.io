信息产业的规律

  1. 摩尔定律：指数级增长的处理能力
  2. 安迪-比尔定律：硬件创造的性能由软件去吃掉
  3. 反摩尔定律：利润会越来越低
  4. 70-20-10律：类似自然界的成王败寇，因为边界成本很低，与传统行业不同。
  5. Norvig定律：一家公司的市场占有率50%以后，就无法再翻番了。
  6. 基因决定定律：受创始人和第一批员工影响很大，企业文化（做事方式）、管理方法、产品市场定位、商业模式和营销模式。

投资银行

  • 美国建立了一个至今看来还非常完善的政治体系，但没有建立这样一个金融体系。

  • 科技公司背后，左右它们发展，影响其商业行为的金融力量：风险投资（未上市）；投资公司或投资银行（已上市）。

  • 1907年 需要一个中央银行，建立 美联储-联邦储备银行系统（Federal Reserve System）

  • 摩根士丹利；高盛；美林；雷曼兄弟；贝尔斯登——存活：摩根和高盛（2008）。

  • 商业银行：接收存款、商业贷款和房屋贷款，如四大银行。

  • 投资公司：不接收存款也不能向中央银行借钱，主要替客户买卖有价证券、期货、不动产和任何有价值的商品。

  • 共同基金公司（Mutual Fund）：为了投资而不是炒作，稳很重要。

  • 对冲基金（Hedge Fund）：不出于投资而是专门炒作的基金，如量子基金。

  • 私募基金和风险投资基金：

  • 资产管理公司：主要做一些资产配置工作，比如把钱交给投资公司去管理，自己也做一些私募基金的事情。

公司的上市过程
  • 风投公司要收回投资，科技公司的创始人和早期员工要得到创业的回报，两条路：被收购和上市。
  • 上市要找承销商，承销商收佣金，通常是包销股票总金额（融资额）的7%；后续以上市价继续购买公司一定股份的权利（期权，只有股价上涨才有意义）
公司的并购
  • 公司的并购和拆分，华尔街都能拿到佣金和期权，但他们本身就有其中两家的股票，甚至是庄家，因此，他们更希望并购和拆分后的结果是可以赚钱的。

生产关系的革命

  • 生产关系：所有权、经营过程中人与人的关系，以及利益的分配制度。

    • 工业时代，企业主和员工的零和游戏。
    • 信息时代，知识变得值钱，不需要企业主驱逐，有知识的员工就先走了。
    • 期权的价值，一家公司到明年的股票价格。

  • 工程师文化：Do It Yourself.DIY，自己实现自己的想法，而不是动口不动手。

    • 第一等工程师：开创行业
    • 第二等工程师：改变世界
    • 第三等工程师：行业最优
    • 第四等工程师：领导产品
    • 第五等工程师：独立完成任务
    • 助理或技工：只是懂一点工程实现的手段，需要别人告诉他怎么做。

  • 扁平式管理

    • 层级少
    • 分权
    • 限制上级对下级的人事权

  • 轻资产，少土地等固定资产

最佳的商业模式：印钞机

  • 挣钱是我们生存的手段还是我们生活的目的。

  • 商业模式要简单明了，一眼可见。

  • Google的广告系统

    • Google提供的服务大多是免费的，但是它的收入在哪里呢？

  • eBay的电子商务系统

    • eBay——淘宝、闲鱼，PayPal——支付宝，亚马逊——天猫商城

  • 戴尔的PC直销系统

    • 只把握订单和销售两个环节，其他的制造、仓储等全部外包。

  • 腾讯的虚拟物品和服务

    • 发行Q币

  • 不好的商业模式

    • 每增加一份营收就必须多雇佣一个人；
    • 无法横向拓展的业务，例如从一个地区扩张到另一个地区要做许多额外的工作；
    • 需要消耗过多的原料和成本；
    • 依赖政府的政策。

互联网2.0（1999——2019年），资源的优化配置。

  • 炒作概念，概念你能弄懂吗？concept

  • 互联网1.0时候，门户网站风行（新浪、搜狐、网易），信息是单向流入的，网民想要有发言权，唯一的途径是自建网站；女王想分享自己的生活，只能联系媒体来进行发布。这时候更像是一个传统的媒体、传统的电视台上了网。BBS留言板只能围绕主题展开讨论，并且不是自己的空间。

  • 2000-2002年，互联网泡沫崩溃，不过这种崩溃是对于投资者和创业者来说的。Google开发了它的广告系统，从而为上亿用户提供免费服务。

  • 互联网2.0 的特征

    • 一个平台，可以接受并且管理用户提交的内容，而这些内容是服务的主体，比如youtube和bilibili。
    • 一个平台，允许用户在平台上开发自己的应用程序，并且提供给其他用户使用。例如微信的小程序。
    • 交互性，目前还未实现。
    • 非竞争性与自足性。用户提供、拥有和享用，一种自给自足的生态环境。

  • 互联网2.0的公司

    • bloger（博客）：个人可以向公众发表言论，及时和方便。
    • 2006年的twitter，微博客。
    • 维基百科（Wikipedia）
    • YouTube
    • Facebook
    • 2006年的twitter（微博客）：没有清晰的商业模式，不好盈利。新浪微博和腾讯微博。
    • 微信：具备互联网2.0的全部特征。

  • 2000——2020年，互联网2.0的这20年，促进了分工协作。

  • 2014——2020年，移动互联网异军突起。

金融风暴的冲击

  • 2008年 金融危机，风险增大，所有人都持现金保留财富，流通不足，只能政府救市。
  • 政府救市不仅会扭转货币流通性不足，还会形成流通性过剩，催生泡沫。
  • 要想走出危机、关键是靠下一次的技术革命！
  • 2010年 欧洲债务危机，高福利政策
  • 经济和财富逐步影响和决定政治地位，用财富去交换一些封建特权。
  • 2009年——2019年，美国持续稳定发展，失业率创下3%的新低。
    • 一个年轻、开放的移民国家。
    • 一个倡导自由市场经济和全球自由贸易的国家。
    • 金融业十分发达，货币政策、金融政策和信用制度非常好。
    • 民众有多大意愿和行动去创造财富，而不是享受福利。
  • 2018年——至今 中美贸易战
  • 2019年 英国正式启动推出欧盟

云计算，真正的革命，科技发展大浪潮！

  • 云计算究竟是什么？它已经证明了它的魔力。

    • 首先，云计算保证用户可以随时随地访问和处理信息，非常方便地与他人共享信息。
    • 其次，云计算保证用户可以使用云端的大量计算资源，包括CPU处理器和储存器（内存和硬盘），无须自己购置设备。

  • 云计算的缘起

    • 微软——客户端（client）操作系统和软件，甲骨文——服务器端（server）数据库。因此甲骨文希望直接通过网络服务替代微软的本地客户端服务，但是由于摩尔定律和网速等现实原因的局限，失败了。
    • 2005年 Google、IMB、Amazon都提出云计算的概念，一个是要把用户的应用搬到网上，一个要卖设备，一个希望向商家和网站出售计算能力。Google Earth将客户端上的软件搬上web上面，将doc Photoshop等都搬上了web上，Gmail、calendar都搬到了网上。

  • 云计算的核心技术

    • 储存：海量数据的存储、结构化数据的存储。Google File Syetem和BigTable
    • 资源管理：Mapreduce和brog
    • 信息安全：配套的政策和法规。
    • 基础：全球基础架构-大型数据中心的建设，全球高速光纤主干网的铺设，制冷技术等。
    • 统一的接口标准和服务，这一点有赖于开放来解决，比如Android。

新的IT（Information Technology）产业链

  • 1990——2012年，以WinTel为主，现在则不一样了，一切都向云端靠拢。
  • 开发个人电脑软件的公司和开发者，他们会将大部分功能从PC端转移到云端，在电脑或手机端只留下简单的操作界面和网络通信功能。体量很重的PC软件就被简化成了一个浏览器的插件（微信小程序）或只能手机上的App。这使得网络浏览器和手机App商店变得十分重要。
  • 亚马逊AWS（Amazon Web Services）最成功的云计算平台，大量的企业选择了它，逐步从微软的下游公司转为AWS的下游。
  • 中国的云计算产业，2009-2013年炒作概念，泡沫破裂后，真正有实力的留了下来，阿里云、腾讯云等。
  • 软件即服务，把软件放在云计算中心，以服务的形式提供给顾客使用。

工业革命和颠覆式创新的范式

  • 人类所经历的三次完整的工业革命，目前是第四次，信息革命伴随着能量和信息不同工作形式。

    • 工业革命以前的物质在2000年内都不会发生重大的变化。
    • 18世纪60年代-19世纪40年代（1760—1840），第一次工业革命：蒸汽机，机器代替人力和畜力。核心资源：煤和铁。
    • 1870—1914年，第二次工业革命：电灯，电力大规模应用，动力的传输，流水线；内燃机的发明。核心资源：石油。
    • 1946—（1980PC）—至今，第三次工业革命（信息化革命）：电子计算机；核能。核心资源：资本。
    • 2016——至今，第四次工业革命（人工智能）：AlphaGo。核心资源：数据。
    • 100年左右的周期

  • 工业革命的范式：现有产业+新技术=新的产业

    • 现有产业+蒸汽机=新的产业
    • 现有产业+电=新的产业
    • 现有产业+信息技术（摩尔定律）=新的产业
      • 农业+信息技术=孟山都种子公司
      • 金融业+信息技术=LED屏报价，通过电话和网络成交。

  • 第四次工业革命

    • 周期基础：距离1950年左右已经70年了，摩尔定律所支持的物理机（基于半导体材料的集成电路）性能已经快到了极限。
    • 理论基础：机器学习的理论基础。在物理机一样的情况下，几十倍地提高计算机解决问题的能力。
    • 现有产业+智能技术=新的产业

信息时代的科学基础

  • 从机械论到“三论”：控制论、系统论、信息论。

    • 牛顿给人类信心，机械论，拉普拉斯妖。确定性和可预知性。
    • 1856-1915泰勒《科学管理原理》；1861-1919 甘特完善理论。
      • 效率优先。//1801年发行可替换式标准零件，1901年发明流水组装生产线。
      • 优化流程与标准化管理。//工厂是个大机器，工人是大机器上的一个零件。
      • 甘特图
      • 苏联使用计划部门来计划经济，预测市场，效果不佳，根本无法做到。
      • 人性化管理：胡萝卜+大棒管理。劳资双方的利润分配是个零和游戏。

  • 1948年Norbert Wiener《控制论》；1948年Claude Shannon 香农《通信的数学原理》《继电器和开关电路的信号分析》；1948年《生命问题》系统论负责制造原子弹。

  • 控制论的思维方式：应变而非预测

    • 炮弹发射，根据牛顿力学原理，如果能准确考虑所有的因素，炮弹的落点应该在目标附近，这就是典型的机械思维方式。问题是风向、湿度等等等所有因素，你是不可能完全考虑清楚的。
    • 火箭登月，准确达到预定的登陆地点，这是如何做到的呢？前者要对未来做一种尽可能确定的预测，而后者则是根据变化不断进行调整。

  • 信息论的思维方式：建立在不确定之上，开放而非封闭

    • 任何一个封闭的系统永远会朝着熵增加（也就是越来越无序）的方向发展，一定会越来越糟糕，只有开放系统，与外界交换，才能更好发展。
    • 信息时代，谁掌握了信息，谁就能发大财。
    • 香农第一定律
    • 香农第二定律：任何时候信息传播得速率都不可能超过通信信道的能力，即带宽。

  • 系统论的思维方式：整体而非局部

    • 每一个部分都做到最优，然后组装起来，整体必然达到最佳状态。
    • 整体的性能未必能通过局部性能的优化而实现。

  • 信息时代思维指南

    • 预测VS反应：未来不可预测，快速反应。
      • 《卓有成效的管理者》每一个知识工作者本身就是一个自觉的自我管理者，对他们不能也不需要采用过去那种简单的自上而下的人事管理方式，而要改成任务导向的契约式管理方式。

    • 宽容失败，其实是对不确定性的认可。
    • 期权背后的科学：从存量分配变为增量分配。

  • 大数据和互联网思维的本质

    • 大数据思维的科学基础：信息论，大量强相关数据的涌现，改变了人们的思维方式和工作方法，这就是大数据思维。二战前和二战期间，信息量的突然暴增：

      • 各种无线电信号信息，里面既有通信信息，也有雷达主动探测收到的信息，它们直接导致了信息论的诞生。
      • 研制武器和使用武器过程中产生的大量数据，比如设计和控制火炮的各种数据，它直接导致了电子计算机的出现和控制论的诞生。

      • 利用信息解决不确定性问题。
