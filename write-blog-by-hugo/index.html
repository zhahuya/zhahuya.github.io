什么是Markdown？

Markdown是一种通过少量简单的标记字符来格式化文本的方法。您可以用纯文本来书写文档，然后在阅读时呈现一个美观的排版。

Markdown的特点

  • 纯文本编辑，简单优雅。
  • 多平台支持，迁移性强。
  • 语法简单，极其易学。

幕布—条理清晰的大纲笔记

”谁能拒绝Markdown？”幕布是一个很有竞争力的选手，它让我们完全无须关注任何格式，完全地集中于内容的创作，而且它还支持一键转换成思维导图，也支持导出PDF、HTML等格式，用来日常使用，记些读书笔记大纲、突然袭来的灵感、整理年月日计划挺不错的，你可以点击这里，赠送1个月的高级版，但它的缺点就在于没有格式，我想要一个一次写成，可以全平台都能发布的文章，它就不是那么适合了。

使用Markdown的前提（个人经验）

  1. 你要对键盘很熟悉，码字速度起码达到45WPM以上，这时候你会觉得鼠标是个累赘，所以一种更科学的调整格式的方式“Markdown”就应运而生了，使用它写出来的排版是那么的美观，且能全平台迁移，谁能拒绝Markdown呢？
  2. 如果你有在各个平台发布文章的需求，比如简书、知乎、张大妈等，你就只能选择Markdown了，不然幕布是个非常好的选择。

使用Markdown的一些建议

  • 4号标题是比较中规中矩的，123号酌情使用。
  • 段与段之间最好隔一行，挨得太近可能会有冲突，而空一行是毫无影响的。如果要换行，必须使用字符来进行提示。
  • 尽量减少标题的层数，这会使后面的文段缩进，看起来不整齐。
  • 规范的使用标点，一串句号看起来很蠢。。。。。。

Markdown编辑工具推荐

我的要求

  1. 开源，不但因为开源的一般是免费的，还因为开源的要更好用。
  2. 要小，只专注于编辑，其他乱七八糟的功能都不要。
  3. 精美，不要花里胡哨的美，而是朴素的精致。

Vnote

  1. 经过了typora，tea，yu write，最后选择了Vnote，完全符合我的要求不说，强大的程度也完全超出了我的预期。
  2. 导出为PDF文件，可以下载推荐的插件，导出文件更美观。
  3. 导出为HTML后，使用Power CHM来制作chm小册子。
  4. 导出自定义文档，使用Pandoc工具（配置环境变量）来生成各种文档。

  6. 这是一位大佬专门为此录制的使用视频，按需观看

日常使用

  1. 在Vnote中写作完成后，将文件放入contents中
  2. 可以使用hugo server进行预览
  3. 生成public文件： hugo --theme=pure --baseUrl="https://zhahuya.github.io/"
  4. 在public文件夹将更新push到云端
  5. 将Hugo源文件push到云端
  6. 使用Onedrive将这一套环境（hugo+vnote）打包，随处可用。
