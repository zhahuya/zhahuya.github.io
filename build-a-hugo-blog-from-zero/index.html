前言

  • 一个虚拟主机，一个域名，缺一些静态文件，使用 hugo 整点静态文件放上去……
  • 此次基于 hugo 的 even 主题搭建。
  • 厌倦了 typecho、WordPress 的发布、编写，只想批量地把 .md 文件渲染了……

    wordpress 的古腾堡编辑器确实无比强大，此次更新又增加了许多功能，但却很不简单。你可以费很大心思在编辑器上面，你会获得回报，但我认为心思更应该放在文章的内容上。强大（powerful）和简单（simple）在这里有了冲突，所以这不是一个好设计（idea），只有强大和简单同时具备的事物，才是值得吹嘘的事物。

清空过去

  • 除了文章原文件外，删除所有以前的痕迹，hexo 的，hugo 的，一切从零开始。

hugo 的安装与配置

  • 下载 hugo 最新版本。hugo 一直在疯狂的更新之中，而 even 主题已经很少更新了，但好像还在做一些新版本的适配工作……
  • 配置环境变量：右键 此计算机 - 属性 - 高级系统设置 - 环境变量 - Path 将 hugo 所在的路径添加到里面。
    • 我的路径是 D:\hugo_Windows-64bit ，没有写版本号，这意味着如果要更新 hugo，直接把最新的 hugo 程序丢到这个文件夹里就好了。
  • 使用 hugo version 来确认你已经安装配置好了hugo，顺便可以查看 hugo 版本。

创建博客，添加主题，调试博客

  • hugo new site Myblog-hugo 创建一个名为“Myblog-hugo”的文件夹
  • git init 将其设置为一个 Git 仓库
  • git clone git@github.com:olOwOlo/hugo-theme-even.git themes/even
  • ：git pull 进行主题的更新或者手动去更新，建议手动更新，直接把 .git 删除来管理主题。
  • 在主题的 exampleSite 目录下有一个 config.toml 文件，将这个 config.toml 文件复制到你的站点目录下，根据自己的需求更改。
  • 你可以通过 front-matter 针对每一篇文章单独进行设置。themes/even/archetypes/default.md 文件陈列了所有可用的参数。

    参考文档：even 主题的使用说明 作者的教程：快速开始文档

  • hugo server 来预览。hugo server -D 草稿也预览。
  • 使用hugo 就可以直接生成 public 文件。theme 和 baseURL 可以在 config.toml 直接指定 。

使用 git 同步站点文件

  • 在 GitHub 上创建一个新仓库用来管理站点文件
  • 在本地”Myblog-hugo”文件夹下新建一个 .gitignore 文件，内容：public/ themes/，用来忽略这两个文件夹。

  • 将themes/even使用 onedrive 或者坚果云或者任何什么东西来保存。

使用 git 管理 public 文件

  • 在 GitHub 上创建一个新仓库，名称必须是 yourname.github.io，这样才可以直接访问这个名称来访问 GitHub Pages。
  • 在“public”文件夹下：git init
  • git remote add origin git@github.com:zhahuya/zhahuya.github.io.git
  • git add .
  • git commit -m "first submit"
  • git push -u origin master

为 git 设置代理

windows 下，找到 .ssh 文件夹 新建一个 config 文件，输入：

Host github.com ProxyCommand "C:\Program Files\Git\mingw64\bin\connect.exe" -S 127.0.0.1:7891 %h %p

参考：在 windows 上为 git ssh 设置代理

同步的问题

~~- 现在我们使用 Git 来同步整个网站目录（“Myblog”文件夹），这样主题 even 只能以 submodule 的形式获取和同步，public/ 则是使用 gitignore 来忽略，其实 public/ 也是一个 git 仓库。

  • 这样我不能对 even 本身有任何改动，因为到了另一台电脑上，我只能获取原本仓库的主题文件，对主题文件的改动就丢失了。不去改动原主题文件，这一点其实是 hugo 所倡导的，它的文件结构也说明了这一点。~~

- 另一个思路：使用 onedrive 或“坚果云”来同步整个网站目录（”Myblog“文件夹），主题文件和 public 文件就各是各的正常的 git 仓库，这样还可以保证整个站点目录的完整性。

日常操作

  1. 在 Vnote 中写作完成后，将文件放入 content/post 中
  2. 使用hugo server进行预览
  3. 生成public文件： hugo
  4. 将站点文件 push 到云端
  5. 将 public/ 文件 push 到云端
  6. 对 even 的改动使用 onedrive 单独保存

使用腾讯云静态网站托管发布网页

  1. 开通你的云环境
  2. 安装 cloudbase cli：npm install -g @cloudbase/cli，你应该先安装node.js，如果你切换了源，那么应该使用cnpm install -g @cloudbase/cli，cli 是 command-line interface 的缩写。
  3. 登陆：tcb login
  4. 将你的网站文件部署到云端：cloudbase hosting:deploy ./public e EnvID

    参考腾讯文档：静态托管部署

改善 even 主题

作者的文档：快速开始

even 主题的优点

  1. 设计优秀：全页面一片纯色、浑然一体，简洁大方。向各种方块说不（Jane和loveit主题）。
  2. 得体、易用的文章目录，~~缺陷一支持的层级太浅，只到3级题目，~~二样式不如（loveIt）美观；向花里胡哨的目录说不（Jane主题）。

even 主题的不足

  1. 【已解决】文章目录支持的层级太浅，只支持到3级标题；目录的样式是 Markdown 中无序列表的样式，不美观。
  2. 【已解决】社交分享模块图标黑又大，笨重，难看，无法全部去除，会留下RSS图标。
  3. 无内置搜索
  4. 无流行的暗黑主题
  5. 不支持文章分享

一款异常优秀的主题：LoveIt

  1. 一款同样干净、简洁、优雅、强大、活跃的主题。

调试 LoveIt 主题

  1. 将 config 中的位置设置为：themesDir = "./themes/"
  2. 删除 English 和 franch 语言，主要是还需要搭配该语言的文章才能有内容，没有必要弄这个。
  3. 使用 hugo_extended 来使 调试 CSS 样式变得可行。
  4. 全页面一片纯色：在./assets/css/.variables.scss 将 global 和 header 的背景颜色改为：白色#fefefe and 黑色#252627，修改标题栏高度：$header-height-desktop: 4.5rem !default;
  5. 调整 header title and manu 的字体与位置：在./assets/css/_partial/_header.scss 添加修改：header {width: 80%;} .header-title {font-family: rage; width: 60%;}
