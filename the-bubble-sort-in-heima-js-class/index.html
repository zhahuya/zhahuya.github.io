前面的话

冒泡排序是一个简单的排序算法，经常拿来作为初学者教学使用，但彻底理解它还是要花点功夫的，这对初学者而言并不简单。

此外还有两个因素：一是在课堂上，有些老师在教学时会受限于自己的认知或者因为太熟悉而无法想象初学者的困惑点，导致出现细节上的纰漏；二是自学的时候，在网络上搜索“冒泡排序”，所获取的信息良莠不齐，更加大了初学者学习的难度。

今天，作为一个初学者，我采用 1+1=2 的笨方法来理解一下这个算法。

本文以此视频为基础介绍：黑马JS教程，冒泡排序算法

文章到最后是一个反转：教会我知识的课程，最后我用自己的思考校正了课程的不合理之处，并获得了更多，这就是不盲从，独立思考吧！说实话作为一个初学者要跳出老师的槽臼，相信自己的直觉挺难的，我做到了(〃‘▽’〃)。

对冒泡排序的想法

  • 题目要求：将数组 arr1 = [3,4,5,2,1] ，从小到大排序。

  1. 针对这个数组，可以看到共有5个数，即 arr1.length = 5 ，如果这个数组有 N 个数，N 就是 arr1.length。

  2. 我们要把这5个数从小到大各归其位，【从第一个数字3开始：3和4比较、不动；4和5比较、不动；5和2比较、换位；5和1比较、换位；这样就筛选出了一个最大的数5】。

    • 即解决了一个最大数的问题，接下来是解决倒数第二的数、倒数第三的数、倒数第四的数，当有4个数的位置被确定时，第五个数的位置也就确定了，因为就剩它一个了，就只能在那了。
    • 因此如果一个数组的长度是 N ，那么要通过这种【两两比较】所要解决的次数就是 N-1。为了和下面的次数区分，我们把这个叫做趟数，即需要 N-1 趟。

  3. 将每一趟筛选数的方法单独抽出来看：

    • 第一趟-两两比较了4次：3和4比较、不动；4和5比较、不动；5和2比较、换位；5和1比较、换位。结果arr1 = [3,4,2,1,5]
    • 第二趟-两两比较了3次：3和4比较、不动；4和2比较、换位；4和1比较、换位；4和5无需比较。结果arr1 = [3,2,1,4,5]
    • 第三趟-两两比较了2次：3和2比较、换位；3和1比较、换位；3和4无需比较；4和5无需比较。结果arr1 = [2,1,3,4,5]
    • 第四趟-两两比较了1次：2和1比较、换位；2和3无需比较；3和4无需比较；4和5无需比较。结果arr1 = [1,2,3,4,5]
    • 找规律：如果一个数组的长度是 N ，那么每趟要比较的次数就是 N - 趟数

var arr1 = [3, 4, 5, 2, 1];
for (var i = 1; i < arr1.length; i++) { // 外层循环管趟数,执行n-1趟
    for (var j = 0; j <= arr1.length - i - 1; j++) { // 里面循环管每一趟的交换次数,执行n-i次
        // 前一个元素和后一个元素比较,如果前>后,则交换两个变量的值.
        if (arr1[j] > arr1[j + 1]) {
            var temp = arr1[j];
            arr1[j] = arr1[j + 1];
            arr1[j + 1] = temp;
        }
    }
}
console.log(arr1);

关于第3步的想法：

  1. 每一趟比较都确定了一个数，因此下一次比较时都不需要再去比较已经确定的数，因此【需要比较的元素个数】随着 趟数的增加 递减1，这是动态的关系。若从1开始计数，关系式为：（N-i+1）
  2. 【需要比较的次数】=【需要比较的元素个数】- 1，这是固定的关系。一个数学事实，3个数比大小，两两比较2次，即可完成排序。
  3. 最后若 i 从 1 开始计数，【趟数】和【需要比较的次数】关系为：N - i + 1 - 1

1 5 4

3 3 2
4 2 1

    for (var j = 0; j <= arr1.length - i - 1; j++) { // 里层循环管每一趟的交换次数

        }
    }

console.log(arr1);

  1. 外层循环管趟数：5个数要排4趟，即var i = 1; i<= arr1.length - 1; i++，因为数组的索引从0开始，因此写为：var i = 0; i < arr1.length -1; i++ 等同于 var i = 0; i <= arr1.length - 2; i++，只有这样，可以保证 i 可取的值为 n-1个。

    • 有 1 个错误（已更正）：应为 var i = 0; i < arr1.length -1; i++, 去掉 = 号, 才是4趟。
    • 有 1 个令初学者疑惑的重要干扰项：外层管趟数，无需从 0 开始，从 1 开始就好了，从0开始计数大大增加理解这个简单算法的难度。

  2. 内层循环管每一趟的交换次数，次数就是N - 趟数，即var j = 1; j <= arr1.length -i; j++，因为数组的索引从0开始，因此写为var j = 0; j <= arr1.length -i - 1; j++
    - 这里，j 能不能从1开始取值，是可以的，只要我们在写交换代码的时候，写成j-1 和 j比较即可。

  3. 在内层循环中写出两数比较的交换代码。

  4. 使用双重 for 循环 来实现冒泡排序。

疑问

  1. j 从 0 开始是可以理解的，因为 j 要在数组中取值，那么 i 为什么需要从 0 开始呢？
    答：因为 j 和 i 是联动的，如果 j 从 0 开始, 而 i 从 1 开始, 那么第一趟的次数n-i次, 就会是4次，咦，这没有什么影响啊！？？！！
    代码试一下，果然输出是正确的，也就是说我们可以写成上面的代码。
