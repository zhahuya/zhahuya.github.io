关于腾讯云的静态网站托管服务

2020-05-08
日常生活
约 1109 字 预计阅读 3 分钟

文章目录

之前在知乎看到了腾讯云的静态网站托管服务，觉得非常方便，就想把自己的小博客放在这里，然而我的经历说明了转到这里需谨慎，这只是一个半成品，如果你和我一样技术不好，还是再等等，功能完善了再来。

三个问题：

  1. 备案，这是一个很大的拦路虎，只是一点小随笔而已，就是要备案，当然这不是腾讯的错，但确实是我们需要考虑的问题。
  2. 无法强制将 http 访问转换为 https 。
  3. 无法将 abc.com 重定向到 www.abc.com。

客服回复：

尊敬的客户， 您好，与您电话沟通，已经明白您的需求，目前这边处理方式：
1.关于http强制转成https ，目前应该是不支持强转的， 关于这块的功能，这边也会反馈到我们产品侧，做对应的评估， 后续您这边继续关注产品动态即可。
2. abc.com 自动重定向到www.abc.com ，该功能是由域名解析这块提供，您可以看看域名解析相关规则，若能解析成功，那云开发侧这边应该也是能够使用的。该功能，我这边也会反馈到产品这边， 等待产品研发侧评估。
已经收到您的需求，这边会反馈给我们后端进行评估，您的建议是我们改善的方向和前进的动力。因此也希望您多体验体验我们的产品，有任何的需求或者建议都可以向我们反馈，也请您能给我们时间来成长完善，谢谢您！

解决方式：

  1. 关于第一个问题，我尝试了很多方法，但是技术不行，失败。
  2. 关于第二个问题，使用 Cname 解析了一下，也是不行，可能是因为我技术不行吧，失败。
  3. 目前只能放弃！
  4. 更新解决方式：在域名解析那里，使用 显性URL，重定向到一个完整的 https://www.abc.com 这种网址上，很好的解决方式。

一些想法：

这个半成品的功能浪费了我许多时间，我烦的很，与客服沟通后我也想了一下甲方和乙方思维的不同，用户希望这个功能可以直接实现他的需求，但是企业那边却是想这个安全证书问题归 SSL 那边管，这个域名重定向问题归域名解析那边管，可以说这是大公司的通病了，想想备受批评的 zf 打太极踢皮球，不也是这个问题吗，分工越精细，协调就越重要，不过令我诧异的是腾讯这种企业也有这样的问题，我觉得不应该。

最后的更新：

我要批评一下自己，有“等靠要”的思想，在仔细看了客服的回复以后，我到域名解析那里看了一下，成功完成了301的重定向。

不过也不必太责怪自己，这是因为技术还不特别扎实的原因，需要日后继续的学习；另外企业提供的服务确实要考虑到方方面面，但这并不是我们就可以不去思考的借口，作为一个有志于学习计算机方面的人来说，应该自己动手去解决问题。（PS 把问题写下来是解决问题很好的一种方式）

我想记录一点生活，但是假如是技术方面的话，这个时间有疑问，等到那个时间点疑问解除了，我得修改文章，保证技术的正确，那我记录生活的作用就没了，比如这篇，我的开篇是“还是不能使用”，现在我必须要修改“已经可以使用了”，烦！
